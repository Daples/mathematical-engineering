Programming Lab 1: Simulating Pushdown Automata

TA: Juan Sebastián Cárdenas-Rodríguez

Introduction

Finite automata only represent regular languages, hence it is important to know a model that allow to represent non-regular languages. One such model are the Pushdown Automaton (PDA) [Hopcroft, Motwani and Ullman 2007]. The goal of this programming lab is to simulate a PDA.

PDA can be deterministic or non-deterministic. Furthermore, PDA can accept by final state or accept by empty stack. In this programming lab, the non-deterministic PDA that accept by final state are going to be considered [Sections 6.1.1 - 6.2.1, Hopcroft, Motwani and Ullman 2007].

Assignment (70%)

  • Download the GitHub repository 2020-1-lab1.

  • (60%) Using the data type in the above repository define the function:

     simPDA :: PushDownAutomata state symbol ssymbol -> [symbol] -> Bool

    that takes a pushdown automaton and a list of symbols and returns if the string of symbols belong or does not belong to the language of the automaton.

    Suggestion: Use the Stack data type.

  • (10%) To document (in English) your source code. Documentation should explain your solution for the assignment.

Examples

Using the PushDownAutomata.hs module in the above GitHub repository, this is what a sample input-output looks like:

      GHCi> simPDA pda1 []
      True

      GHCi> simPDA pda1 [1]
      False

      GHCi> simPDA pda1 [0,1,1,0]
      True

      GHCi> simPDA pda1 [0,1,1,1,0,0,1,1,1,0]
      True

      GHCi> simPDA pda2 "aaaab"
      False

      GHCi> simPDA pda2 "aabb"
      True

      GHCi> simPDA pda3 "aabb"
      False

      GHCi> simPDA pda3 "aaaaaabbbbbbbbb"
      True
